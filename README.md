# Automated Geo-Routing & VPN Infrastructure Manager

Данный репозиторий содержит комплекс Python- и Bash-скриптов для автоматизации работы с сетевыми маршрутами, агрегации CIDR-списков и их автоматического деплоя на сетевое оборудование. Реализовано на примере роутеров KeeneticOS.

Проект разработан для управления трафиком в распределенной VPN-инфраструктуре (на базе серверов в Амстердаме и Алматы). Инструментарий позволяет гибко настраивать политики маршрутизации, управлять доступом к внутренним ресурсам и автоматизировать обновление конфигураций для WireGuard и AmneziaWG.

## 🛠 Стек технологий

*   **Языки:** Python 3.11, Bash
*   **Сетевое оборудование:** KeeneticOS, Entware (Linux)
*   **VPN / Proxy:** WireGuard, AmneziaWG
*   **Ключевые библиотеки:** `ipaddress`, `paramiko`, `geoip2`, `urllib`
*   **Базы данных:** GeoLite2 (MaxMind)

## 🚀 Архитектура и основные компоненты

Репозиторий состоит из четырёх независимых, но логически связанных модулей:

### 1. Гео-балансировка и фильтрация — `GeoRouting`
Модуль отвечает за географическое разделение трафика между удаленными узлами.
*   Использует локальную базу данных `GeoLite2-Country.mmdb` для быстрой проверки принадлежности IP-адресов.
*   Определяет принадлежность подсетей к целевой зоне (страны СНГ и ближнего зарубежья) для маршрутизации через сервер в Алматы.
*   Остальной трафик автоматически перенаправляется через сервер в Амстердаме.
*   Формирует итоговый файл конфигурации `routing_rules.txt` с разбивкой по тегам `##AMS` и `##ALM`.

### 2. Агрегация и SSH-деплой — `KeenRoute`
Ядро автоматизации для взаимодействия с сетевым оборудованием.
*   Динамически загружает актуальные списки IPv4-адресов из публичных CDN и специализированных репозиториев.
*   Выполняет строгую валидацию и агрегацию CIDR (collapse addresses) с помощью встроенного модуля `ipaddress` для минимизации нагрузки на таблицу маршрутизации роутера.
*   Реализует загрузку данных по SSH (через `paramiko`) напрямую в среду Entware с последующим автоматическим перезапуском службы маршрутизации.
*   Для KeeneticOS списки автоматически разбиваются на группы лимитом до 300 строк и привязываются к интерфейсам WireGuard через FQDN-объекты.

### 3. Конвертация форматов и интеграция — `TorrentCIDRData`
Утилита для обеспечения совместимости с клиентским программным обеспечением.
*   Подключается к роутеру по SSH и извлекает актуальный `ip.list`.
*   Генерирует JSON-конфигурации для клиентских приложений AmneziaWG с корректным экранированием спецсимволов.
*   Создает `ipfilter4torrents.dat` для блокировки или разрешения специфичного трафика на уровне торрент-клиентов.

### 4. Мониторинг и резервное копирование — `BashScripts`
Два независимых скрипта для Entware/KeeneticOS.

**`rtk_bleach.sh`** — hook-скрипт, запускаемый при изменении состояния PPPoE-интерфейса:
*   Проверяет текущий публичный IP-адрес, полученный от провайдера по PPPoE.
*   Определяет, не попал ли адрес в диапазон Carrier-Grade NAT (100.64.0.0/10) или другие приватные диапазоны — то есть не выдал ли провайдер "серый" адрес вместо настоящего публичного.
*   При обнаружении CGNAT автоматически инициирует переподключение PPPoE (до 10 попыток), чтобы получить реальный публичный IP.
*   Уведомляет о результате через локальный syslog (`logger`) или Telegram Bot API — способ уведомления переключается флагом в скрипте.

**`ent_back.sh`** — периодическое резервное копирование содержимого Entware (запускается по cron):
*   Архивирует содержимое `/opt` в tar.gz с именем, включающим идентификатор устройства и дату создания.
*   Отправляет готовый архив в Telegram через Bot API (`sendDocument`).
*   Если архив превышает 50 МБ (лимит Telegram на отправку документов), отправка пропускается, а в чат уходит уведомление о необходимости забрать бэкап вручную по SFTP.
*   После успешной отправки удаляет локальную копию архива, чтобы не занимать место на роутере; ход выполнения логируется через `logger`.

## ⚙️ Установка и использование

### Требования
```bash
pip install paramiko geoip2
```

### Клонирование

```bash
git clone https://github.com/TIIGR/network_engineering.git
cd network_engineering
```

### Запуск модулей

**GeoRouting** — распределение CIDR-списков по гео-зонам:
```bash
python -m GeoRouting.geo
```

**KeenRoute** — агрегация списков и деплой на роутер по SSH (пароли запрашиваются интерактивно, не хранятся в коде):
```bash
python -m KeenRoute.script
```

**TorrentCIDRData** — конвертация актуального списка с роутера в форматы для AmneziaWG и торрент-клиентов:
```bash
python TorrentCIDRData/convert_ip.py
```

**BashScripts** — два скрипта с разным способом запуска:

`rtk_bleach.sh` устанавливается как hook-скрипт Entware, отслеживающий события WAN-интерфейса:
```bash
cp BashScripts/rtk_bleach.sh /opt/etc/ndm/wan.d/
chmod +x /opt/etc/ndm/wan.d/rtk_bleach.sh
```

`ent_back.sh` запускается по расписанию через cron (например, раз в неделю):
```bash
cp BashScripts/ent_back.sh /opt/etc/ent_backup/
chmod +x /opt/etc/ent_backup/ent_back.sh
# добавить в crontab, например:
# 0 4 * * 0 /opt/etc/ent_backup/ent_back.sh
```

Перед использованием обоих скриптов задайте `BOT_TOKEN` и `GROUP_CHAT_ID` внутри скрипта для уведомлений через Telegram, либо оставьте локальное логирование через `logger`.

## 🔒 Безопасность

Все скрипты, работающие с SSH, запрашивают учётные данные интерактивно через `getpass` — пароли не хранятся в коде и не попадают в репозиторий.
